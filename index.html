Teach a Computer to Solve Puzzles

How Do We Teach a Computer to Escape a Maze?

Imagine a simple 2D maze puzzle:

  • There is a Start point
  • There is anEnd/Goal point
  • A few Traps placed that you want to avoid

Your challenge is to program a robot to solve this maze and reach from Start to Goal. You could try to manually code rules like “avoid traps” or “move toward the goal,” but this quickly becomes messy and doesn’t scale if the maze changes.

A Better Idea: Let the Robot Learn by Itself

Instead of manually specifying rules, we can let the robot explore the maze, try different things, and remember what works. Thats it!!

If it moves up and falls into a trap → that was bad.
If it moves right and gets closer to the goal → that was good.

So the computer needs a way to keep track of what was good and what was bad.

To explain this further, let me introduce a couple of terms: State, and Action

A State just means, "where you are right now."
In a maze, each box or tile is a state (row 1 col 2 or row 3 col 4).

An Action is simple what you choose to do next.

For example:

  • Move up
  • Move down
  • Move left
  • Move right

Some times when you collect coins, you will get bonus score, or if you step on a trap, you loose points. So, lets call this Reward. Its positive when robot does something good - like finishing puzzle or find coins on the way. It will be negative if it falls in a trap or gets eliminated.

For example:

  • Reaching the end goal: + 100 points
  • Stepping on a trap: -100 points
  • Each normal move: -1 points (energy used to move)

Our robot tries out the maze a few times with mixed results and remembers what happened last time when it went in a certain direction. In other words, it has an expectation of what reward it gets if it takes a certain action in a given state. Lets call this expected reward as Q-value.

For example:

  • If you are in row 2, col 3 (state) and you go left (action), you will hit a trap -> -100 points (reward)
  • If you are in row 2, col 3 (state) and you go up (action), you will hit energy boost -> +100 points (reward)

Thats it.. no more new jargon!

When the robot enters the maze for the first time, it does not have any expectation of what might happen from any action. Which means, Q-value is zero for all state action pairs. So, it takes random actions.

But, as it keeps exploring the maze, whenever it encounters a good or bad state (either positive or negative reward), it immediately updates the Q-value for the state action pair which led to this state.

In other words, After you try something, you adjust your expectations based on what actually happened. And in future, when you are in a similar situation, follow the path with highest expected Q-value

Thats it. Now you understand how Q-learning works!! If you want to know more details about the exact math behind it, you can continue reading. Or else go to the demo.

The following is the exact equation that is used to calculate the updated Q-value by using an experience of robot.

$$ Q(s, a) ← Q(s, a) + \alpha [ r + \gamma max Q(s', a') - Q(s, a) ] $$

Whoa.. thats a lot of math. Before you panic, lets explain it in plain words.

New score = Old score + How much the current experience change our mind about expectation.

That “How much the experience changed our mind” part is:

  • What reward we got right now
  • PLUS the best possible score we can get in the future
  • MINUS what we originally thought would happen

The symbols:

  • \( Q(s, a) \): The expected Q-value
  • \( r \): the reward we got for the specific action
  • \( max_{a'} Q(s', a') \): best expectation from future state
  • \( \alpha \): how fast we learn from new information
  • \( \gamma \): how much we care about future

You can see the same equation in action in the demonstration.
